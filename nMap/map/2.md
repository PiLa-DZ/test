# Phase 2: Scanning & Vulnerability Assessment

Target IP (`192.168.0.185`) from Phase 1,
you are going to switch your strategy.
Instead of looking at the whole network,
you will hyper-focus on this single device.

The goals of this phase are to find **all open ports**,
determine **exactly what services and versions** are running behind them,
identify the **Operating System**, and check for **known vulnerabilities**.

Here are the essential commands you need to execute on your Arch Linux machine,
ordered from basic scanning to advanced vulnerability identification.

---

## 1. Full Port Scanning (Finding the Doors)

Before identifying vulnerabilities,
you must scan the entire port range to ensure no hidden services are missed.

- **Scan all 65,535 TCP ports aggressively:**

```bash
sudo nmap -p- --min-rate 2000 192.168.0.185

```

_Why:_ As you saw in your earlier test,
a standard scan misses high-number custom ports (like `3770`).
This forces Nmap to check every single possible entry point quickly.

---

## 2. Service Version & OS Detection (Banner Grabbing)

Once you know which ports are open (let's say port `3770` or port `80`),
you need to figure out _what_ software is listening on that port
and _what_ OS the target uses.

- **Detect Service Versions and OS Fingerprint:**

```bash
sudo nmap -p 3770 -sV -O 192.168.0.185

```

- `-sV`:
  Service Version detection. It sends specific probes to the open port
  to make the service reveal its exact software and version
  (e.g., `Node.js Express 4.18.2` or `Apache httpd 2.4.41`).

- `-O`:
  Operating System detection.
  It analyzes how the target's network stack responds to judge
  whether it is running Linux, Android, Windows, or an embedded RTOS.

- **The "All-in-One" Advanced Scan:**

```bash
sudo nmap -p 3770 -A 192.168.0.185

```

- `-A`:
  Enables OS detection, version detection, script scanning (`traceroute`),
  and default intelligence scripts all at once.

---

## 3. Vulnerability Scanning (Finding the Cracks)

Now that you have the service names and versions,
you transition into automated vulnerability assessment.
Nmap has a powerful built-in engine called **NSE (Nmap Scripting Engine)**
that can automatically map service versions against known exploit databases.

- **Scan for known vulnerabilities using the Vuln script category:**

```bash
sudo nmap -p 3770 --script vuln 192.168.0.185

```

_Why:_ This checks the open ports against
a massive local database of known CVEs (Common Vulnerabilities and Exposures).
If your service is outdated, Nmap will print out the exact vulnerability name,
risk score, and pre-written exploit references.

- **Fast Vulners Database Lookup:**

```bash
sudo nmap -p 3770 --script vulners -sV 192.168.0.185

```

_Why:_ This queries real-time security databases.
It looks at the version string returned by `-sV`
and prints out a tidy list of links to public exploits
(like Exploit-DB entries) available for that specific version.

## Phase 2 Cheat Sheet Summary Table

| Command                             | Operational Goal                | Key Flags Used               |
| ----------------------------------- | ------------------------------- | ---------------------------- |
| `nmap -p- --min-rate 2000 <IP>`     | Find every open port            | `-p-` (All 65k ports)        |
| `nmap -p <port> -sV -O <IP>`        | Identify specific software & OS | `-sV` (Version), `-O` (OS)   |
| `nmap -p <port> -A <IP>`            | Aggressive deep scan            | `-A` (Combo flag)            |
| `nmap -p <port> --script vuln <IP>` | Look for known security flaws   | `--script vuln` (NSE engine) |
