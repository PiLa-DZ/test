# Phase 1: Reconnaissance (Information Gathering)

- **Your Step 1 (Find one device / Discovering IPs):**
  This belongs right here! Using `nmap -sn 192.168.0.1/24`
  to sweep the network and discover that your target is sitting
  at `192.168.0.185` is the textbook definition of local active reconnaissance.

---

## Phase 2: Scanning & Vulnerability Assessment

- This phase is divided into two distinct sub-steps:
  - **Network Scanning** (finding the doors)
  - **Vulnerability Assessment** (finding the cracks in those doors).

- **Your Step 2 (Port Scanning):**
  Running `sudo nmap -p- 192.168.0.185` to find open ports
  (like port `3770`) is the network scanning portion of this phase.

- **Your Step 3 (Gathering OS, Device name, Services):**
  This is also part of scanning,
  often called **Banner Grabbing** or **Service/OS Detection**. In Nmap,
  you do this using the `-A` or `-sV` and `-O` flags (e.g., `nmap -sV -O 192.168.0.185`).
  This is how you pulled the "Infinix" or "Midea" manufacturer signatures from the MAC addresses.

- **Your "Phase 2"
  Note (Finding Vulnerabilities):**
  This is the core of Vulnerability Assessment.
  Once you know the device runs _Node.js v18_ or _Windows 7_,
  you map that specific version against a database of known vulnerabilities
  (like CVEs) using automated scripts or websites.

---

## Phase 3: Exploitation

This matches your understanding perfectly.
Now that you have a list of vulnerabilities,
you attempt to drop a payload or run an attacking script to bypass security.

- **How this works in your simulation:**  
   If your Step 3 revealed that a target machine is running an ancient,
  unpatched service with a known Remote Code Execution (RCE) flaw,
  you would open an exploitation framework like **Metasploit** on Arch Linux.

- You would search for the exploit module matching that vulnerability,
  set the payload (like a reverse shell),
  point it at the target IP, and hit exploit.

- If it succeeds,
  your Arch terminal transforms into a root terminal inside the victim's machine.
