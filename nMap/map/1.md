# Targeting a local network (`LAN`)

Active Network Discovery

---

## 1. Network Mapping & Subnet Identification

Before you can scan anything,
you need to know your own machine's IP address
and the range of your home network (the subnet).

- **Check your own local IP and network interface:**

```bash
ip a

```

_Look for your wireless interface (usually `wlan0` or `wlp2s0`)
to see your local IP (e.g., `192.168.0.108`)._

- **Find your default gateway (Router IP):**

```bash
ip route show | grep default

```

_This confirms your router's IP address (e.g., `192.168.0.1`),
which defines your network range as `192.168.0.0/24`._

---

## 2. Host Discovery (Finding the Devices)

The goal here is to send a quick "ping" to all 256 possible addresses
on your network to see who answers.
This helps you discover the IPs of the unknown devices without touching their ports yet.

- **The Standard Ping Sweep (Fastest):**

```bash
nmap -sn 192.168.0.0/24

```

_`-sn` tells Nmap to skip port scanning completely. It just checks if the hosts are alive._

- **The ARP Scan (Highly Accurate on local Wi-Fi):**

```bash
sudo arp-scan --localnet
# _or using Nmap:_
sudo nmap -PR -sn 192.168.0.0/24

```

_`-PR` forces Nmap to use ARP requests. On a local network,
devices cannot hide from ARP requests;
even if they have a local firewall blocking standard pings,
they **must** reply to ARP to stay connected to the Wi-Fi._

---

## 3. Passive Asset Discovery (Listening Quietly)

Instead of shouting into the network with a scan,
you can sit back and listen to the packets floating through the air to discover devices passively.

- **Check your local ARP cache table:**

```bash
ip neigh show

```

_This instantly prints a list of every device your Arch Linux laptop
has recently talked to or noticed on the network,
along with its MAC address, without sending a single probe._

- **Sniff network broadcast traffic (Live Capture):**

```bash
sudo tcpdump -i wlan0 -n icmp or arp

```

_This keeps your terminal open, listening live.
If an unknown device connects to the Wi-Fi or tries to talk to the router,
its IP and MAC address will instantly flash across your screen._

## Cheat Sheet Summary Table for Phase 1

| Command                      | Purpose             | Why it's useful                                                 |
| ---------------------------- | ------------------- | --------------------------------------------------------------- |
| `ip route show`              | Find Router Gateway | Identifies your subnet target range.                            |
| `nmap -sn <subnet>`          | Ping Sweep          | Tells you exactly how many devices are turned on.               |
| `sudo nmap -PR -sn <subnet>` | ARP Discovery Sweep | Bypasses local device firewalls to reveal hidden IPs.           |
| `ip neigh show`              | View ARP Cache      | Instantly shows known devices without generating network noise. |
